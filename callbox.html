Import Csv
Instruction
  • 1. When you Import a file if a file region already exists then replace data in that region
    • - Case 1 Example: If System has test_region in the database, we import test_region.csv file, all test_region.csv call box data updated in the system.
    • - When you insert a CSV file then in its data there is a column name (ANI). if ANI column data is present in the database then its data is updated and its status set is active. if ANI column data present in the database does not contain in the CSV file then its status is set to inactive. if ANI column data in CSV file is absent in database then it is inserted in the database and its status is set to active.
    • For example test_region.csv
      • ANI - 5678 is in a database and also in CSV file then its record will be updated and its status will be set to active
      • ANI - 6789 is in the database and it is absent in the CSV file then its status is set to inactive.
      • ANI - 9879 is absent in the database and it is preset in CSV file then its record will be inserted and its status is set to active
  • 2. if a file region does not exist then that file region is created and all CSV data is created on that region
    • - Case 2 Example: If we import test_region_new.csv file, the system does not have test_region_new region then the region is created by the system. Data with test_region_new region are imported to call box data for the newly created region from the CSV file.
  • 3. region names are case sensitive if in database no matching region found then system auto-creating a new region.
    • - metrolink and Metrolink are different region
    • - region selection is not consider in import region. if you select metrolink region and try to import Metrolink.csv data then system first check Metrolink region if that region not exist then auto-creating a new region and import all csv data in that region.
